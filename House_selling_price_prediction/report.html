Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

    """ Calculates and returns the performance score between 
        true and predicted values based on the metric chosen. """

Answer: The goal of fitting an algorithm is to create a general model for classification/regression. The goal is not to fit to all the points in the data set but to learn the general trends from the data so that the fitted model can be applied for data outside of the current set.

If we use the entire data as training set to fit a model there would be no way to check if the model is memorizing the data or is extracting the pattern. If the training set and test set both have high error we can infer that we have taken too simplistic model (high bias). On the other hand if the training error is small but test error is large then we have high variance which signifies that the model is overfitted. We can look at the train and test errors to understand if the model is under or overfitted.

Question 7 - Grid Search¶

  • How it can be applied to optimize a learning algorithm?

Hint: When explaining the Grid Search technique, be sure to touch upon why it is used, what the 'grid' entails and what the end goal of this method is. To solidify your answer, you can also give an example of a parameter in a model that can be optimized using this approach.

Answer:

The grid search technique tunes hyperparameters of a model in order to get the optimal performance.

The performance of any machine learning algorithm depends on several hyperparameters. For example DecisionTreeClassifier has max_depth as one of the hyperparameters. Given a data set we can optimize the hyperparameters to best fit the data in this set. A grid is a set of combinations of the hyperparameters which we evaluate our model for. Once we know the performance of our model on all of these combinations we can choose the optimal one for the data. For this we split data into train-validate-test sets in some ratio (say 50%-30%-20%). Then we keep the test set separate. We fit our model for each combination of hyperparameters on the training set and note the performance metric on the validation score. Once we have convered the entire 'grid' of hyperparameter combinations we choose the combination corresponding to the best validation set score to get optimal model. We then test our model on the test set.

  • What benefit does this technique provide for grid search when optimizing a model?

When thinking about how k-fold cross validation helps grid search, think about the main drawbacks of grid search which are hinged upon using a particular subset of data for training or testing and how k-fold cv could help alleviate that. You can refer to the docs for your answer.

Answer:

In k-fold cross-validation method the data is randomly divided into k equal sized small subsets. Then we keep one of the subset as our validation set and we execute training on other k-1 subsets. The whole process of validation and training is repeated k times. Each of the k subsets acts as test set once. The average of the results from k steps is used as the estimate of the overall performance of the model.

The grid search method can efficiently find the optimal the hyper parameter for a set of train and test set. But if we do not use the k-fold cross validation, the parameters get tuned only for those specific sets of data which is not a reliable estimation of the hyperparameters in general and there is high chance that grid search will select the hyper-parameter values that perform very well on a specific train-test split but poorly otherwise. So in summary, the k-fold cross validation helps the grid search method to find more reliable or general set of hyper parameters.

from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV
from sklearn.model_selection import ShuffleSplit

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    #cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)
    cv_sets = ShuffleSplit(n_splits=10, test_size=0.20, train_size=None, random_state=0)
    

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth':range(1,10)}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)#creates a pointer to the performance metric so that GridSearchCV can call it

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor,params,scoring=scoring_fnc,cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

Sensitivity¶
